# Trajectory Deviation Dataset Pipeline

This repository contains a small, reproducible pipeline for building a trajectory-deviation video dataset and a static website for browsing the resulting manifest.

The target clips show a person following an intended plan, then visibly diverging from it in a way where a streaming assistant could help them recover.

## Quick Start

```powershell
python -m pipeline.run_pipeline --candidates data/candidates.csv --out site/data/dataset.json --report build/report.md
python -m http.server 8000 -d site
```

Then open http://localhost:8000.

## Publish at `/streaming_dataset/`

Create a GitHub repository named `streaming_dataset`, push this project to it, and enable **Settings > Pages > Source > GitHub Actions**. The included workflow publishes the `site/` folder, so the public URL will be:

```text
https://<your-github-username>.github.io/streaming_dataset/
```

## Project Layout

- `data/source_map.json` - seed source families and academic dataset pointers.
- `data/candidates.csv` - editable candidate clip metadata.
- `pipeline/` - cue scoring, normalization, and manifest generation.
- `site/` - static dataset browser that reads `site/data/dataset.json`.
- `build/` - generated reports.

## Candidate CSV Schema

Add one row per raw clip candidate in `data/candidates.csv` with:

```csv
id,title,description,transcript,url,source_family,domain,goal,start_seconds,deviation_onset_seconds,end_seconds,license,license_status,hosting_status,notes
```

The CSV is only a candidate queue. Final validity, deviation labels, visual evidence, timing, and assistant responses are generated by the VLM verifier.

See `data/curation_guide.md` for the step-by-step curation process.

## VLM Verification With Gemini

Set your API key:

```powershell
$env:GEMINI_API_KEY = "your-key"
```

Verify candidates:

```powershell
python -m pipeline.gemini_verify --candidates data/candidates.csv --out data/verifications.json
```

Regenerate the public dataset:

```powershell
python -m pipeline.run_pipeline --candidates data/candidates.csv --verifications data/verifications.json --out site/data/dataset.json --report build/report.md --verified-only
```

Without `data/verifications.json`, candidates appear as `awaiting_vlm` and are not treated as curated.

## Auto-Collect Candidate URLs

Add YouTube channels to `data/channels.csv`, then run:

```powershell
python -m pipeline.collect_candidates --channels data/channels.csv --candidates data/candidates.csv
```

By default, the collector reads each channel's public RSS feed, scans recent titles and descriptions for deviation cues, and appends only new URLs. Set `include_all` to `true` for a channel if you want Gemini to inspect every recent upload.

The full automated loop is:

```powershell
python -m pipeline.collect_candidates --channels data/channels.csv --candidates data/candidates.csv
python -m pipeline.gemini_verify --candidates data/candidates.csv --out data/verifications.json
python -m pipeline.run_pipeline --candidates data/candidates.csv --verifications data/verifications.json --out site/data/dataset.json --report build/report.md --verified-only
```

The public site is built with `--verified-only`, so rejected and awaiting candidates stay in the private working files but do not appear in `site/data/dataset.json`.

## Dataset Framing

The benchmark task is:

> Detect when the user's observed trajectory is diverging from their intended plan, and produce a timely intervention that helps them recover.

The generated manifest contains source cards plus candidate clip records with cue scores, inferred deviation types, intervention windows, and assistant-message drafts.
